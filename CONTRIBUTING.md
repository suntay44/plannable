# Contributing to Plannable

Thanks for helping! Plannable is a zero-runtime-dependency TypeScript CLI — please keep it that way.

## Setup

Requires Node.js 18+.

```bash
git clone https://github.com/suntay44/Plannable.git
cd Plannable
npm install
npm run build
npm test
```

## Development loop

```bash
npm run dev -- create "CRM"   # run the CLI from source
npm run typecheck             # strict TypeScript, no emit
npm test                      # vitest: CLI integration + core unit tests
```

## Project layout

```txt
src/cli.ts        argument dispatch, version, friendly errors
src/commands/     one file per CLI command (thin shells)
src/core/         parsing, rendering, state regeneration (all logic lives here)
templates/        MASTER_PLAN / PLAN_STATE / PLAN_EVIDENCE / PART_PLAN sources
tests/            cli.test.ts (end-to-end in temp dirs), core.test.ts (unit)
.agents/ .codex/ .claude/ .cursor/   platform skill files
```

## Rules

- **No runtime dependencies.** Node builtins only; devDependencies are fine.
- **MASTER_PLAN.md + PLAN_EVIDENCE.md are the source of truth.** `PLAN_STATE.md` is generated by `regenerateState()` — never patch it line-by-line.
- **Skill files stay aligned.** `.codex/skills/plannable/SKILL.md` must be byte-identical to `.agents/skills/plannable/SKILL.md` (a test enforces this). Apply shared changes to all four platform SKILL.md files.
- **The format is called PlannablePlan**, never PlanPack. `@PlannablePlan v0.1` only bumps on breaking changes; parsers must ignore unknown blocks.
- **Tests for behavior changes.** End-to-end CLI tests go in `tests/cli.test.ts` (temp dirs, no mocks); pure-logic tests go in `tests/core.test.ts`.

## Submitting a change

1. Fork, branch from `main`.
2. Make the change with tests; run `npm run typecheck && npm test`.
3. Open a PR describing what changed and why. Update CHANGELOG.md under `[Unreleased]`.
